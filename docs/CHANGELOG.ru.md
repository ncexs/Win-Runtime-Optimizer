## 🚀 [2.0.0-Auto] - 2026-05-10

### 🛠️ Исправления ошибок (Bug Fixes)
- **Safe-SetService Fix:** Исправлена критическая опечатка, из-за которой в версии v1.0.0 вызывалась функция `Safe-SetService`, но была определена как `Set-ServiceStartupSafely`. Ранее это приводило к тихому сбою всех оптимизаций запуска служб Windows.

### ✨ Новые функции (New Features)
- **Integrated Self-Updater:** Скрипт теперь может автоматически определять, загружать, применять и перезагружать последнюю версию напрямую из репозитория GitHub (`ncexs/Win-Runtime-Optimizer`) при запуске.
- **Расширенная поддержка приложений:** Добавлена безопасная очистка кэша для **Mozilla Firefox**, **Opera GX** (игровой браузер) и **Discord** (популярное приложение для общения).
- **GPU & Shader Cache Cleanup:** Добавлена физическая очистка папок кэша шейдеров для графических процессоров **NVIDIA** (DXCache, GLCache, ComputeCache), **AMD** (DxCache, OglCache) и **Intel** (ShaderCache).
- **Расширенные параметры запуска:** Введены мощные элементы управления параметрами:
  - `-Silent` (работает полностью в фоновом режиме без вывода в консоль)
  - `-SkipUpdate` (пропускает начальную проверку автообновления при запуске)
  - `-ForceUpdate` (принудительно перезагружает скрипт с GitHub)
  - `-CustomBranch` (назначает пользовательскую ветку релизов на GitHub)

### 🔒 Безопасность и стабильность (Security & Stability)
- **Автоматическое определение неинтерактивного режима (`UserInteractive`):** При выполнении через Task Scheduler или фоновые службы скрипт автоматически определяет неинтерактивную среду и включает тихий режим (`$Silent = $true`). Это предотвращает зависание скрипта из-за ожидания ввода пользователя (например, `Read-Host` во время обновления).
- **Умная очистка с учетом активных процессов:** Проверка браузеров и приложений теперь выполняется точечно. Если Google Chrome активен, а Firefox закрыт, скрипт безопасно пропускает Chrome и очищает Firefox (ранее пропускались все браузеры глобально, если хотя бы один из них был активен).
- **Полная совместимость с ASCII:** Все псевдографические символы Юникода в консольных баннерах и логах заменены на 100% стандартные символы ASCII. Это предотвращает сбои парсинга/кодировки в устаревших средах Windows PowerShell 5.1.

## 📦 [1.0.0] - 2026-05-10

### ✨ Первый релиз (Initial Release)
Первый выпуск портативного, безопасного, тихого и стабильного скрипта для обслуживания среды выполнения Windows.

- Добавлен автоматический сброс лог-файла при каждом запуске скрипта.
- Очистка кэша браузеров обновлена для учета активных процессов.
- Из очистки браузеров удалены GPUCache и ShaderCache для предотвращения визуальных сбоев (glitch) в приложениях на базе Chromium.
- Устранены состояния гонки (race conditions), которые вызывали искажение отображения в браузере до ручного обновления страницы.
- Внутренние вспомогательные функции переименованы с использованием утвержденных глаголов PowerShell (`Is-` -> `Test-`) для полного соответствия рекомендациям PSScriptAnalyzer.
- Внедрено автоматическое определение оборудования и системы (Windows 10/11, объем ОЗУ и определение дисков SSD/HDD).
- Очистка Prefetch настроена только для дисков HDD (для предотвращения избыточного износа SSD).
- Реализована адаптивная настройка запуска служб Windows (SysMain, WSearch, DiagTrack, Xbox и ClipSVC).
- Разработана стратегия уменьшения рабочего набора памяти (memory trim) специально для систем с малым объемом ОЗУ (<= 8 ГБ) без влияния на активные веб-браузеры.
- Добавлено безопасное и полное удаление загруженных установочных файлов Windows Update.
- Интегрировано структурированное логирование с метками времени в файл.
- Обеспечена абсолютная совместимость скрипта для автоматического фонового запуска через Task Scheduler каждые 6 часов.

**Результаты релиза 1.0.0:**
- Никаких искажений дисплея или визуальных сбоев в браузерах.
- Абсолютная стабильность для активных сессий WhatsApp Web.
- Безопасные и бесперебойные запланированные фоновые запуски.
