# ⚡ ncexs Auto Task (NAT)

🌐 [English](../README.md) | [Bahasa Indonesia](README.id.md) | [Basa Jawa](README.jv.md) | [Basa Sunda](README.su.md) | **हिन्दी** | [Русский](README.ru.md) | [日本語](README.ja.md) | [한국어](README.ko.md) | [简体中文](README.zh.md) | [العربية](README.ar.md)

📄 **[बदलाव सूची](CHANGELOG.hi.md)** | 🛠️ **v2.0.0-Auto**

**ncexs Auto Task** एक पोर्टेबल PowerShell स्क्रिप्ट है जिसे आपके सक्रिय एप्लिकेशन या ब्राउज़र सत्रों को बाधित किए बिना सुरक्षित, मौन (silent) और स्थिर Windows रनटाइम रखरखाव और अनुकूलन के लिए डिज़ाइन किया गया है।

## ▶️ कैसे चलाएं

### मैन्युअल निष्पादन

PowerShell चलाएं (यदि इंटरैक्टिव मोड में चलाया जाता है तो यह स्वचालित रूप से व्यवस्थापक अधिकारों के लिए संकेत देगा) और निम्नलिखित आदेश चलाएं:

```powershell
powershell -ExecutionPolicy Bypass -File ncexs-AutoTask.ps1
```

#### ⚙️ वैकल्पिक पैरामीटर (उन्नत)

इन मापदंडों का उपयोग करके निष्पादन को कस्टमाइज़ करें:

* **`-Silent`**: किसी भी कंसोल आउटपुट के बिना स्क्रिप्ट को पूरी तरह से बैकग्राउंड में चलाता है। Task Scheduler के लिए अनुशंसित।
* **`-SkipUpdate`**: स्टार्टअप पर स्वचालित अपडेट जांच (सेल्फ़-अपडेटर) को छोड़ देता है।
* **`-ForceUpdate`**: स्थानीय संस्करण के समान होने पर भी स्क्रिप्ट को GitHub रिपॉजिटरी से फिर से डाउनलोड करने के लिए बाध्य करता है।
* **`-CustomBranch "branch_name"`**: अपडेट जांचने के लिए एक कस्टम ब्रांच सेट करता है (डिफ़ॉल्ट `"main"` है)।

उपयोग का उदाहरण:
```powershell
powershell -ExecutionPolicy Bypass -File ncexs-AutoTask.ps1 -Silent -SkipUpdate
```

## ⚙️ उन्नत मार्गदर्शिकाएँ और सेटअप

<details>
<summary><b>⏱️ स्वचालित सेटअप (Task Scheduler) मार्गदर्शिका देखने के लिए क्लिक करें</b></summary>

### 1️⃣ Task Scheduler खोलें
**Win + R** दबाएं, टाइप करें:
```
taskschd.msc
```
**Enter** दबाएं।

---

### 2️⃣ एक नया कार्य बनाएँ
**Create Task** पर क्लिक करें (*Basic Task* का चयन न करें)

**General टैब**
- Name: `ncexs Auto Task`
- जांचें:
  - Run whether user is logged on or not
  - Run with highest privileges

---

### 3️⃣ Triggers टैब
- **New** पर क्लिक करें
- कार्य शुरू करें: **On a schedule**
- **Daily** चुनें

**उन्नत सेटिंग्स (वैकल्पिक):**
- कार्य दोहराएं: **6 hours**  
  *(या यदि आप इसे रोजाना एक बार चलाना चाहते हैं तो इसे अनचेक छोड़ दें)*
- अवधि के लिए: **Indefinitely**
- **Enabled** जांचें
- **OK** पर क्लिक करें

> ⚠️ ड्रॉपडाउन मेनू अधिकतम 1 घंटे दिखाता है, लेकिन आप मैन्युअल रूप से **6 hours** या **06:00:00** टाइप कर सकते हैं। हर घंटे स्क्रिप्ट चलाना **अनुशंसित नहीं है** क्योंकि यह एक हल्का रखरखाव उपकरण है, न कि रीयल-टाइम मॉनिटर।

---

### 4️⃣ Actions टैब
- **New** पर क्लिक करें
- Action: **Start a program**
- Program/script: `powershell.exe`
- तर्क जोड़ें:
  ```
  -ExecutionPolicy Bypass -File "C:\Path\To\ncexs-AutoTask.ps1" -Silent
  ```
- Start in (वैकल्पिक): `C:\Path\To`

> ⚠️ `C:\Path\To` को उस सटीक फ़ोल्डर पथ से बदलें जहाँ आपने स्क्रिप्ट सहेजी है।

---

### 5️⃣ Conditions टैब
- अनचेक करें: `Start the task only if the computer is on AC power`

---

### 6️⃣ Settings टैब
- जांचें:
  - Allow task to be run on demand
  - Run task as soon as possible after a scheduled start is missed
- अनचेक करें: `Stop the task if it runs longer than`

**OK** पर क्लिक करें, और संकेत मिलने पर अपना Windows पासवर्ड दर्ज करें।
> ⚠️ बिना पासवर्ड के कार्य चलाने के लिए, उपयोगकर्ता को **SYSTEM** पर सेट करें (*Change User or Group...* पर क्लिक करें, **SYSTEM** टाइप करें, और OK पर क्लिक करें)।

</details>

<details>
<summary><b>🔒 Task Scheduler में सुरक्षा और स्थिरता सुविधाएँ देखने के लिए क्लिक करें</b></summary>

यह स्क्रिप्ट बैकग्राउंड में **Task Scheduler** के माध्यम से 100% सुरक्षित और बिना किसी सहायता के चलने के लिए डिज़ाइन की गई है:

1. **गैर-इंटरैक्टिव ऑटो-डिटेक्शन (`UserInteractive`):** यदि Task Scheduler द्वारा निष्पादित किया जाता है (भले ही आप `-Silent` पैरामीटर छोड़ दें), तो स्क्रिप्ट स्वचालित रूप से बैकग्राउंड वातावरण का पता लगाती है और साइलेंट मोड (`$Silent = $true`) लागू करती है। यह अपडेट के दौरान इंटरैक्टिव इनपुट संकेतों (जैसे `Read-Host`) पर स्क्रिप्ट को रुकने या फ़्रीज़ होने से रोकता है।
2. **प्रोसेस-अवेयर सुरक्षा:** ब्राउज़र या ऐप कैश को साफ़ करने से पहले, स्क्रिप्ट यह सत्यापित करती है कि क्या वे एप्लिकेशन सक्रिय हैं। यदि सक्रिय है, तो सक्रिय सत्रों की सुरक्षा और डेटाबेस भ्रष्टाचार को रोकने के लिए उस विशिष्ट प्रोग्राम के लिए सफ़ाई छोड़ दी जाती है।
3. **ग्रेसफुल एलिवेशन चेक:** यदि कार्य "Highest Privileges" के बिना गलत तरीके से कॉन्फ़िगर किया गया है, तो स्क्रिप्ट अवरुद्ध नहीं होगी या सेशन 0 में छिपे हुए UAC पॉप-अप को ट्रिगर नहीं करेगी। यह लॉग फ़ाइल में एक साफ़ चेतावनी दर्ज करती है और ग्रेसफुल तरीके से बाहर निकल जाती है।
4. **स्मार्ट Windows Update सफ़ाई:** Windows Update सेवा (`wuauserv`) को केवल तभी अस्थायी रूप से रोका जाता है जब वह पहले से चल रही हो, और सफ़ाई के तुरंत बाद पुनः आरंभ कर दी जाती है। यदि यह पहले बंद थी, तो स्क्रिप्ट आपकी सेटिंग्स को बनाए रखती है।

</details>

<details>
<summary><b>📂 सफ़ाई के लिए लक्षित फ़ोल्डर देखने के लिए क्लिक करें</b></summary>

सफ़ाई के लिए केवल निम्नलिखित विशिष्ट फ़ोल्डरों को लक्षित किया जाता है:
- `%TEMP%` (User Temp)
- `C:\Windows\Temp` (System Temp)
- `C:\Windows\Prefetch` (फ़ोल्डर सामग्री, केवल HDD तक सीमित)
- **Windows Update डाउनलोड कैश** (`C:\Windows\SoftwareDistribution\Download`)
- **GPU & Shader Cache** (NVIDIA, AMD, Intel - DXCache, GLCache, ComputeCache, ShaderCache)
- **गैर-महत्वपूर्ण ब्राउज़र और ऐप कैश** (Chrome, Edge, Brave, Opera Stable, Opera GX, Firefox, Discord):
  - Cache & cache2
  - Code Cache & startupCache
  - GPUCache & DawnWebGPUCache
  - ShaderCache, jumpListCache, & thumbnail

प्रमाणीकरण फ़ोल्डर, सक्रिय उपयोगकर्ता सत्र और कुकीज़ **पूरी तरह से अछूते** रहते हैं।

</details>

<details>
<summary><b>🔧 रखरखाव और समीक्षा मार्गदर्शिका देखने के लिए क्लिक करें</b></summary>

यदि भविष्य के अपडेट में सिस्टम फ़ाइलें, OS संरचना, या ब्राउज़र पथ बदलते हैं:
- ब्राउज़र प्रोफ़ाइल और कैश के लिए फ़ोल्डर पथों की समीक्षा करें।
- सुनिश्चित करें कि उपयोगकर्ता डेटा और सत्र संग्रहण पूरी तरह से सुरक्षित रहें।
- जहाँ आवश्यक हो, रनिंग प्रोसेस जांच लॉजिक को समायोजित करें।

</details>
