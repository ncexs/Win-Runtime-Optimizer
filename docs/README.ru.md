**ncexs Auto Task** — это легкий и портативный скрипт PowerShell, разработанный для безопасного, тихого и стабильного обслуживания и оптимизации среды выполнения Windows без прерывания работы активных приложений или браузерных сессий.

## ▶️ Как запустить

### Ручной запуск

Запустите PowerShell (скрипт автоматически запросит права администратора при интерактивном запуске) и выполните следующую команду:

```powershell
powershell -ExecutionPolicy Bypass -File ncexs-AutoTask.ps1
```

#### ⚙️ Дополнительные параметры (для продвинутых)

Настройте выполнение с помощью следующих параметров:

* **`-Silent`**: Запускает скрипт полностью в фоновом режиме без вывода в консоль. Рекомендуется для Task Scheduler.
* **`-SkipUpdate`**: Пропускает автоматическую проверку обновлений (автообновление) при запуске.
* **`-ForceUpdate`**: Принудительно скачивает скрипт из репозитория GitHub, даже если локальная версия идентична.
* **`-CustomBranch "имя_ветки"`**: Задает пользовательскую ветку для проверки обновлений (по умолчанию `"main"`).

Пример использования:
```powershell
powershell -ExecutionPolicy Bypass -File ncexs-AutoTask.ps1 -Silent -SkipUpdate
```

### 1️⃣ Откройте Task Scheduler
Нажмите **Win + R**, введите:
```
taskschd.msc
```
Нажмите **Enter**.

### 2️⃣ Создайте новую задачу
Нажмите **Create Task** (не выбирайте *Basic Task*)

**Вкладка General**
- Name: `ncexs Auto Task`
- Отметьте:
  - Run whether user is logged on or not
  - Run with highest privileges

---

### 3️⃣ Вкладка Triggers
- Нажмите **New**
- Begin the task: **On a schedule**
- Выберите **Daily**

**Дополнительные параметры (необязательно):**
- Repeat task every: **6 hours**  
  *(или оставьте поле пустым, если хотите запускать задачу один раз в день)*
- For a duration of: **Indefinitely**
- Отметьте **Enabled**
- Нажмите **OK**

> ⚠️ Выпадающее меню планировщика ограничено 1 часом, но вы можете ввести **6 hours** или **06:00:00** вручную. Запуск скрипта каждый час **настоятельно не рекомендуется**, так как это инструмент для легкого обслуживания, а не монитор реального времени.

---

### 4️⃣ Вкладка Actions
- Нажмите **New**
- Action: **Start a program**
- Program/script: `powershell.exe`
- Добавьте аргументы:
  ```
  -ExecutionPolicy Bypass -File "C:\Path\To\ncexs-AutoTask.ps1" -Silent
  ```
- Start in (необязательно): `C:\Path\To`

> ⚠️ Замените `C:\Path\To` на точный путь к папке, в которой сохранен скрипт.

---

### 5️⃣ Вкладка Conditions
- Снимите отметку: `Start the task only if the computer is on AC power`

---

### 6️⃣ Вкладка Settings
- Отметьте:
  - Allow task to be run on demand
  - Run task as soon as possible after a scheduled start is missed
- Снимите отметку: `Stop the task if it runs longer than`

Нажмите **OK** и введите пароль Windows, если появится запрос.
> ⚠️ Чтобы запускать задачу без ввода пароля, укажите пользователя **SYSTEM** (нажмите *Change User or Group...*, введите **SYSTEM** и нажмите OK).

<details>
<summary><b>🔒 Нажмите для просмотра функций безопасности и стабильности в Task Scheduler</b></summary>

Этот скрипт разработан для 100% безопасной автономной работы в фоновом режиме через **Task Scheduler**:

1. **Автоматическое определение неинтерактивного режима (`UserInteractive`):** Если скрипт запускается через Task Scheduler (даже если вы забыли параметр `-Silent`), он автоматически определяет фоновую среду выполнения и включает тихий режим (`$Silent = $true`). Это предотвращает зависание скрипта на запросах ввода (таких как `Read-Host` при обновлении).
2. **Защита с учетом активных процессов:** Перед очисткой кэша браузеров или приложений скрипт проверяет, активны ли они. Если приложение запущено, очистка для него пропускается, чтобы защитить активные сессии и предотвратить повреждение баз данных.
3. **Корректная обработка прав:** Если задача ошибочно настроена без наивысших прав (Highest Privileges), скрипт не заблокируется и не вызовет скрытое окно UAC в Сессии 0. Он запишет предупреждение в лог-файл и корректно завершит работу.
4. **Умная очистка Windows Update:** Служба обновления Windows (`wuauserv`) останавливается временно только в том случае, если она работала до запуска скрипта, и перезапускается сразу после очистки. Если служба была отключена, скрипт сохранит ваши системные настройки.

</details>

<details>
<summary><b>📂 Нажмите для просмотра целевых папок для очистки</b></summary>

Для очистки используются только следующие конкретные папки:
- `%TEMP%` (Временные файлы пользователя)
- `C:\Windows\Temp` (Временные файлы системы)
- `C:\Windows\Prefetch` (Содержимое папки; только для HDD)
- **Кэш загрузок Windows Update** (`C:\Windows\SoftwareDistribution\Download`)
- **GPU & Shader Cache** (NVIDIA, AMD, Intel - DXCache, GLCache, ComputeCache, ShaderCache)
- **Некритический кэш браузеров и приложений** (Chrome, Edge, Brave, Opera Stable, Opera GX, Firefox, Discord):
  - Cache & cache2
  - Code Cache & startupCache
  - GPUCache & DawnWebGPUCache
  - ShaderCache, jumpListCache, & thumbnail

Папки аутентификации, активные пользовательские сессии и Cookies **остаются полностью нетронутыми**.

</details>

<details>
<summary><b>🔧 Нажмите для просмотра руководства по обслуживанию</b></summary>

Если в будущих обновлениях изменятся системные файлы, структура ОС или пути к браузерам:
- Проверьте пути к профилям браузеров и папкам кэша.
- Убедитесь, что пользовательские данные и хранилище сессий остаются под защитой.
- При необходимости скорректируйте логику проверки активных процессов.

</details>

- **Автоматизированный инструмент** с полностью интегрированным механизмом автообновления.
- **Автоматическое версионирование** (начиная с `v1.0.0` как основы первого стабильного релиза).
