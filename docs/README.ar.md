**ncexs Auto Task** هو برنامج نصي خفيف ومحمول لبيئة PowerShell مصمم لصيانة وقت تشغيل Windows وتحسينه بشكل آمن وصامت ومستقر، دون مقاطعة التطبيقات النشطة أو جلسات المتصفح.

## ▶️ كيفية التشغيل

### التنفيذ اليدوي

قم بتشغيل PowerShell (الذي سيطالب تلقائيًا بصلاحيات المسؤول إذا تم تنفيذه في الوضع التفاعلي) وقم بتشغيل الأمر التالي:

```powershell
powershell -ExecutionPolicy Bypass -File ncexs-AutoTask.ps1
```

#### ⚙️ المعلمات الاختيارية (متقدم)

قم بتخصيص التنفيذ باستخدام هذه المعلمات:

* **`-Silent`**: يقوم بتشغيل البرنامج النصي بالكامل في الخلفية بدون أي إخراج على وحدة التحكم. موصى به عند استخدام Task Scheduler.
* **`-SkipUpdate`**: يتخطى فحص التحديث التلقائي الأولي (Self-Updater) عند بدء التشغيل.
* **`-ForceUpdate`**: يجبر البرنامج النصي على إعادة تنزيل نفسه من مستودع GitHub حتى لو كانت النسخة المحلية مطابقة.
* **`-CustomBranch "اسم_الفرع"`**: يحدد فرعًا مخصصًا للتحقق من التحديثات (الافتراضي هو `"main"`).

مثال للاستخدام:
```powershell
powershell -ExecutionPolicy Bypass -File ncexs-AutoTask.ps1 -Silent -SkipUpdate
```

## ⚙️ الإعدادات والأدلة المتقدمة

<details>
<summary><b>⏱️ انقر لعرض دليل الإعداد التلقائي (Task Scheduler)</b></summary>

### 1️⃣ فتح Task Scheduler
اضغط على **Win + R**، واكتب:
```
taskschd.msc
```
واضغط على **Enter**.

---

### 2️⃣ إنشاء مهمة جديدة
انقر فوق **Create Task** (لا تختر *Basic Task*)

**علامة التبويب General**
- الاسم: `ncexs Auto Task`
- حدد:
  - Run whether user is logged on or not
  - Run with highest privileges

---

### 3️⃣ علامة التبويب Triggers
- انقر فوق **New**
- بدء المهمة: **On a schedule**
- حدد **Daily**

**الإعدادات المتقدمة (اختياري):**
- تكرار المهمة كل: **6 hours**  
  *(أو اترك الخيار غير محدد إذا كنت ترغب في تشغيله مرة واحدة يوميًا)*
- لمدة: **Indefinitely**
- حدد **Enabled**
- انقر فوق **OK**

> ⚠️ تظهر القائمة المنسدلة للجدول الزمني بحد أقصى ساعة واحدة، ولكن يمكنك كتابة **6 hours** أو **06:00:00** يدويًا. يُنصح **بشدة بعدم** تشغيل البرنامج النصي كل ساعة لأن هذه أداة صيانة خفيفة وليست مراقبًا في الوقت الفعلي.

---

### 4️⃣ علامة التبويب Actions
- انقر فوق **New**
- الإجراء: **Start a program**
- البرنامج/البرنامج النصي: `powershell.exe`
- إضافة الوسائط:
  ```
  -ExecutionPolicy Bypass -File "C:\Path\To\ncexs-AutoTask.ps1" -Silent
  ```
- البدء في (اختياري): `C:\Path\To`

> ⚠️ استبدل `C:\Path\To` بمسار المجلد الدقيق حيث حفظت البرنامج النصي.

---

### 5️⃣ علامة التبويب Conditions
- ألغِ تحديد: `Start the task only if the computer is on AC power`

---

### 6️⃣ علامة التبويب Settings
- حدد:
  - Allow task to be run on demand
  - Run task as soon as possible after a scheduled start is missed
- ألغِ تحديد: `Stop the task if it runs longer than`

انقر فوق **OK**، وأدخل كلمة مرور Windows عند مطالبتك بذلك.
> ⚠️ لتشغيل المهمة بدون مطالبة بكلمة مرور، قم بتعيين المستخدم على **SYSTEM** (انقر فوق *Change User or Group...*، واكتب **SYSTEM**، وانقر فوق OK).

</details>

<details>
<summary><b>🔒 انقر لعرض ميزات الأمان والاستقرار في Task Scheduler</b></summary>

تم تصميم هذا البرنامج النصي ليعمل بأمان تام بنسبة 100% وبشكل غير مراقب في الخلفية عبر **Task Scheduler**:

1. **الاكتشاف التلقائي للوضع غير التفاعلي (`UserInteractive`):** إذا تم تنفيذه عبر Task Scheduler (حتى إذا نسيت معلمة `-Silent`)، يكتشف البرنامج النصي تلقائيًا بيئة التشغيل في الخلفية ويفرض الوضع الصامت (`$Silent = $true`). هذا يمنع البرنامج النصي من التجمد بسبب انتظار مطالبات الإدخال التفاعلية (مثل `Read-Host` أثناء التحديث).
2. **حماية تراعي العمليات النشطة:** قبل تنظيف ذاكرة التخزين المؤقت للمتصفح أو التطبيق، يتحقق البرنامج النصي مما إذا كانت هذه التطبيقات قيد التشغيل. إذا كانت قيد التشغيل، يتم تخطي التنظيف لهذا البرنامج المحدد لحماية الجلسات النشطة ومنع تلف قواعد البيانات.
3. **التحقق السلس من الصلاحيات:** إذا تمت تهيئة المهمة بشكل خاطئ بدون "أعلى الامتيازات"، فلن يتعطل البرنامج النصي أو يعرض نافذة UAC منبثقة ومخفية في الجلسة 0. بل يقوم بتسجيل تحذير نظيف في ملف السجل ويخرج بسلاسة.
4. **تنظيف ذكي لخدمة Windows Update:** يتم إيقاف خدمة تحديث Windows (`wuauserv`) مؤقتًا فقط إذا كانت قيد التشغيل بالفعل عند بدء البرنامج النصي، ويتم إعادة تشغيلها فور انتهاء التنظيف. أما إذا كانت معطلة أو متوقفة في الأصل، فيحافظ البرنامج النصي على تفضيلات النظام الخاصة بك.

</details>

<details>
<summary><b>📂 انقر لعرض المجلدات المستهدفة بالتنظيف</b></summary>

يتم استهداف المجلدات المحددة التالية فقط للتنظيف:
- `%TEMP%` (الملفات المؤقتة للمستخدم)
- `C:\Windows\Temp` (الملفات المؤقتة للنظام)
- `C:\Windows\Prefetch` (محتويات المجلد فقط، تقتصر على محركات HDD)
- **ذاكرة تخزين تنزيلات Windows Update** (`C:\Windows\SoftwareDistribution\Download`)
- **ذاكرة التخزين المؤقت لوحدة معالجة الرسومات والتظليل GPU & Shader Cache** (NVIDIA, AMD, Intel - DXCache, GLCache, ComputeCache, ShaderCache)
- **ذاكرة التخزين المؤقت غير الحرجة للمتصفحات والتطبيقات** (Chrome, Edge, Brave, Opera Stable, Opera GX, Firefox, Discord):
  - Cache & cache2
  - Code Cache & startupCache
  - GPUCache & DawnWebGPUCache
  - ShaderCache, jumpListCache, & thumbnail

مجلدات المصادقة، وجلسات المستخدم النشطة، وملفات تعريف الارتباط (Cookies) تظل **غير ممسوسة تمامًا**.

</details>

<details>
<summary><b>🔧 انقر لعرض دليل الصيانة والمراجعة</b></summary>

إذا تغيرت ملفات النظام أو بنية نظام التشغيل أو مسارات أدلة المتصفح في التحديثات المستقبلية:
- راجع مسارات الأدلة لملفات تعريف المتصفحات وذاكرة التخزين المؤقت.
- تأكد من بقاء بيانات المستخدم وتخزين الجلسات محمية بالكامل.
- اضبط منطق التحقق من العمليات قيد التشغيل عند الضرورة.

</details>
